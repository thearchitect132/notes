# **1. Philosophy of Contribution**

Tammetry is not a software project.  
It is a **world‑model**, a **continuity system**, and a **civilisational artefact**.

Contributions must therefore:

- preserve **continuity**  
- preserve **invariants**  
- preserve **narrative alignment**  
- preserve **layer alignment**  
- preserve **deep‑time coherence**  
- preserve **sovereignty**  

Every change participates in the same arc:

```
foundations → systems → execution → civilisation
```

# **2. Contribution Principles**

### **2.1 Continuity First**
Every contribution must maintain:

- identity continuity  
- provenance continuity  
- verification continuity  
- invariant continuity  
- narrative continuity  
- TRU continuity  
- layer continuity  

See the **Continuity Spec** for details.

### **2.2 Invariant Preservation**
Invariants are the laws of the system.  
No contribution may violate:

- structural invariants  
- epistemic invariants  
- ethical invariants  
- TRU invariants  

See **Continuity Foundations**.

### **2.3 Deep‑Time Stability**
Contributions must remain meaningful across:

- releases  
- eras  
- civilisations  

This repository is designed for **epochal longevity**.

# **4. Contribution Workflow**

### **4.1 Fork or Branch**
Create a branch:

```
git checkout -b feature/<name>
```

### **4.2 Make Changes**
Ensure:

- continuity preserved  
- invariants preserved  
- narrative aligned  
- layers aligned  

### **4.3 Commit**
Use clear, continuity‑aware messages:

```
git commit -m "Extend continuity foundations with invariant lattice"
```

### **4.4 Push**
Push your branch:

```
git push origin feature/<name>
```

### **4.5 Pull Request**
A PR must include:

- purpose  
- continuity analysis  
- invariant analysis  
- narrative alignment  
- deep‑time justification
